Note API v2

RESTful note-taking API — Cloudflare Workers + MongoDB

v1 · Express · Mongoose · JWT

Overview

A serverless note-taking API deployed on Cloudflare Workers. Built with Express.js and MongoDB (via Mongoose). Features include user authentication with JWT, note CRUD with search, file/folder management, and device-level request logging.

Base URL

http://localhost:3000/{version}

Default version is v1. All API paths below assume this prefix.

Content Type

All requests and responses use application/json.

Quick Start

Prerequisites

  • Node.js 18+
  • MongoDB instance (local or Atlas)
  • Cloudflare account (for deployment)

Authentication

The API uses JWT (JSON Web Token) for protected routes. Tokens are issued via signup and login, and must be sent in the Authorization header.

Header Format

Authorization: Bearer <token>

Auth Middleware

All endpoints marked with Auth require a valid token. The userMiddleware extracts and verifies the token, attaching the decoded payload ({ id, role }) to req.user.

Token Payload

FieldTypeDescription
idObjectIdMongoDB user ID
rolestringUser role: user, admin, or moderator

Token Expiry

Configurable via JWT_EXPIRES_IN env variable (default: 7d).

User Endpoints

Base path: /v1/user

POST /v1/user/signup

Register a new user account.

Request Body

FieldTypeRequiredDescription
namestringYesDisplay name
emailstringYesEmail address (case-insensitive, stored lowercase)
passwordstringYesPassword (hashed with bcryptjs, 12 rounds)

Example

curl -X POST http://localhost:3000/v1/user/signup \
  -H "Content-Type: application/json" \
  -d '{ "name": "Alice", "email": "alice@example.com", "password": "securePass123" }'

Response 201 Created

{
  "success": true,
  "message": "User created successfully",
  "token": "eyJhbGciOiJIUzI1NiIs..."
}

Error Responses

400Missing fields{ "success": false, "message": "Invalid user data" }
409Email already in use{ "success": false, "message": "Email already in use" }
500Server error{ "success": false, "message": "Failed to create user" }
On success, an OTP is generated and linked to the user for future email verification. Token is also set as an httpOnly cookie named token.
POST /v1/user/login

Authenticate and receive a JWT token.

Request Body

FieldTypeRequiredDescription
emailstringYesRegistered email
passwordstringYesAccount password

Example

curl -X POST http://localhost:3000/v1/user/login \
  -H "Content-Type: application/json" \
  -d '{ "email": "alice@example.com", "password": "securePass123" }'

Response 200 OK

{
  "success": true,
  "message": "Login successful",
  "token": "eyJhbGciOiJIUzI1NiIs..."
}

Error Responses

401Invalid credentials{ "success": false, "message": "Invalid credentials" }
500Server error{ "success": false, "message": "Failed to login" }

Note Endpoints

Base path: /v1/note — All endpoints require authentication.

POST /v1/note/note Auth

Create a new note.

Request Body

FieldTypeRequiredDescription
titlestringNo*Note title
contentobjectNo*See content structure below

Content Object Structure

FieldTypeDescription
typestringContent format. One of: text, markdown, html, code, python, javascript, json, csv, xml, sql, yaml, toml, css, java, go, ruby, php, rust, swift, kotlin, csharp, cpp, c, bash, powershell, jsonc, yamlc, tomlc, htmlc, scss, less. Default: text.
textstringActual content body
filesarrayAttached files: { filename, url, type }
imagesarrayAttached images: { filename, url, type }

Example

curl -X POST http://localhost:3000/v1/note/note \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIs..." \
  -d '{
    "title": "Meeting Notes",
    "content": {
      "type": "markdown",
      "text": "# Agenda\n- Review sprint goals\n- Plan next release"
    }
  }'

Response 201 Created

{
  "success": true,
  "message": "Note created successfully",
  "note": {
    "_id": "66a1b2c3d4e5f6...",
    "title": "Meeting Notes",
    "content": { "type": "markdown", "text": "# Agenda\n- Review sprint goals\n- Plan next release", "files": [], "images": [] },
    "user": "66a1b2c3d4e5f6...",
    "createdAt": "2026-01-01T00:00:00.000Z",
    "updatedAt": "2026-01-01T00:00:00.000Z"
  }
}

File / Folder Endpoints

Base path: /v1/file — All endpoints require authentication.

POST /v1/file/folder Auth

Create a new folder. Folder names must be unique per user.

Request Body

FieldTypeRequiredDescription
namestringYesFolder name (must be unique per user)

Note

Any additional fields in the request body are spread into the folder document (e.g., privacy, sharedWith).

Example

curl -X POST http://localhost:3000/v1/file/folder \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIs..." \
  -d '{ "name": "Work Documents" }'

Response 201 Created

{
  "success": true,
  "message": "Folder created successfully",
  "folder": {
    "_id": "66a1b2c3d4e5f6...",
    "folder": {
      "name": "Work Documents",
      "subfolder": null,
      "files": []
    },
    "owner": "66a1b2c3d4e5f6...",
    "sharedWith": [],
    "privacy": "private",
    "isDeleted": false,
    "createdAt": "2026-01-01T00:00:00.000Z",
    "updatedAt": "2026-01-01T00:00:00.000Z"
  }
}

Error Handling

All error responses follow a consistent format:

{
  "success": false,
  "message": "Human-readable error description"
}

HTTP Status codes used:

CodeMeaning
200Success
201Created
400Bad request (missing/invalid fields)
401Unauthorized (missing/expired token)
404Not found
409Conflict (e.g. duplicate email or folder name)
500Internal server error
503Service unavailable (database connection failed)
